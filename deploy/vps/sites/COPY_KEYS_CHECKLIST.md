## Copy these values from each Supabase project

### 1. API URL

Copy the project URL:

- Source in Supabase: `Project Settings -> API -> Project URL`
- Put it into:
  - `NEXT_PUBLIC_SUPABASE_URL=`
  - `SUPABASE_URL=`

Example:

```env
NEXT_PUBLIC_SUPABASE_URL=https://xxwhasnyoswvbfwtjrbv.supabase.co
SUPABASE_URL=https://xxwhasnyoswvbfwtjrbv.supabase.co
```

### 2. Anon key

Copy the anon/public key:

- Source in Supabase: `Project Settings -> API -> anon public`
- Put the same value into:
  - `NEXT_PUBLIC_SUPABASE_ANON_KEY=`
  - `SUPABASE_ANON_KEY=`

Example:

```env
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...
SUPABASE_ANON_KEY=eyJ...
```

### 3. Service role key

Copy the service role key:

- Source in Supabase: `Project Settings -> API -> service_role`
- Put it into:
  - `SUPABASE_SERVICE_ROLE_KEY=`

Important:

- If Supabase shows the value with a prefix like `service_role-...`, remove the `service_role-` prefix and keep only the JWT value when needed.

### 4. Optional direct database connection

You do not need to put the raw PostgreSQL connection string inside `site.env` for the current app/runtime setup.
The running app and worker depend on the API URL and keys above.

## Values not taken from Supabase directly

These values should be generated by you and should be different for each site:

- `AUTH_SESSION_SECRET`
- `SESSION_SECRET`
- `NEXTAUTH_SECRET`
- `CRON_SECRET`

Recommended rule:

- use one long random secret per site for the three session fields
- use another long random secret per site for `CRON_SECRET`

## Values you can usually keep or fill later

If you are not using these immediately, you can leave placeholders for now and fill them before production use:

- `WEB_PUSH_SUBJECT`
- `WEB_PUSH_PRIVATE_KEY`
- `NEXT_PUBLIC_WEB_PUSH_PUBLIC_KEY`
- `WHATSAPP_PHONE_NUMBER_ID`
- `WHATSAPP_ACCESS_TOKEN`
- `WHATSAPP_VERIFY_TOKEN`
- `WHATSAPP_APP_SECRET`
- `META_APP_SECRET`

- `NEXT_PUBLIC_SUPABASE_URL` and `SUPABASE_URL` are identical

- `SUPABASE_SERVICE_ROLE_KEY` belongs to the same project as the URL
- `WHATSAPP_CLIENT_ID` is unique per site
- `WHATSAPP_AUTH_DIR` is unique per site
- `APP_PORT` is unique per site
